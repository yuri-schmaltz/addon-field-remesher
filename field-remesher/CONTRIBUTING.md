# Contribuindo

## Fluxo
- Abra uma issue (ou pegue uma existente do backlog)
- Faça branch: `feature/<id>-<slug>`
- PR com descrição, prints e testes (quando aplicável)

## Padrões
- Python: formatação consistente (ruff/black opcional via CI)
- Evitar dependências pesadas no add-on
- Preferir degradar com warnings em vez de falhar silenciosamente
